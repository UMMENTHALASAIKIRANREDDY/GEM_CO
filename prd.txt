1. Executive Summary
This document defines requirements for the Gemini-to-Copilot Pages Migration Tool — a fully admin-driven utility that migrates Google Gemini conversation history for all users in an organization into their respective Microsoft Copilot Pages. Users are not involved at any step of the process.

The IT admin exports all users' Gemini conversation history in a single operation using Google Vault — Google's enterprise eDiscovery tool available in Workspace Business Plus and Enterprise plans. The migration tool reads the Vault export, processes each user's data in complete isolation, and creates Copilot Pages in each user's M365 account via delegated Microsoft Graph API permissions. Each user logs into Copilot after migration and finds their conversations waiting in the sidebar.

Key principle	The IT admin runs two operations: (1) export from Google Vault, (2) run the migration tool. No user action is required at any point — no Takeout exports, no ZIP submissions, no user notifications during migration.

2. Problem Statement
2.1 The Core Gap
Users on Microsoft 365 Business/Enterprise switching from Google Gemini to Microsoft Copilot have no way to carry their conversation history across. Copilot starts completely empty for every user. Valuable prompts, iterative work, and generated content built up in Gemini becomes inaccessible once the organisation moves to Copilot.

2.2 Why Native Chat History Injection Is Not Possible
Approach	Why It Fails
Microsoft Graph API write to chat history	No write endpoint exists for Copilot conversation history in any API version including beta
Browser automation (Playwright/Puppeteer)	Creates new conversations dated today only; breaks on UI updates; blocked by M365 bot detection
Copilot Studio import	No import feature for external chat history from third-party platforms
Direct database access	Not available to customers; M365 backend is closed infrastructure

2.3 Why Copilot Pages Is the Right Solution
Property	Detail
Appears in Copilot sidebar	Pages show under 'Recent pages' in the Copilot interface — visible on the user's first login after migration
Fully editable by user	User can add notes, annotations, and highlights directly on the page after migration
Ask Copilot from the page	User can ask Copilot follow-up questions about any page without copy-pasting context
Shareable across M365 tenant	Pages can be shared with colleagues via standard M365 permissions
Supported Graph API	Pages created via Microsoft Graph Loop/Pages API — stable, documented, enterprise-supported
Gemini link inline	Each page includes an expandable Gemini original response with a direct link back to the Gemini conversation

3. Data Source — Google Vault
3.1 What Is Google Vault
Google Vault is Google's enterprise eDiscovery, archiving, and data export tool included in Google Workspace Business Plus, Enterprise Standard, and Enterprise Plus plans. It allows IT admins to export data for any user or all users in the domain without any user involvement or notification.

Prerequisite	Google Vault must be enabled for your Workspace plan. Verify access at admin.google.com → Apps → Google Vault. If not available, upgrade to Business Plus or Enterprise Standard before proceeding.

3.2 Why Google Vault Over Other Options
Option	Verdict
Google Vault (eDiscovery)	Recommended — admin exports all users' Gemini history in one operation, no user involvement
Google Workspace Admin Data Export	Available but does NOT include Gemini AI conversation history — covers Gmail, Drive, Calendar only
Google Workspace Admin SDK	No Gemini chat API available — cannot pull conversation history programmatically
Individual user Takeout	Requires each user to act — not suitable for fully admin-driven migration

3.3 How Admin Exports Gemini Data via Google Vault
Step	Action
1. Open Vault	Navigate to vault.google.com — sign in with Google Workspace admin credentials
2. Create a matter	Matters → Create matter → Name: 'Gemini Migration [date]'
3. Create an export	Inside the matter → Exports → Create export
4. Select data type	Data type: Gemini for Google Workspace → Select all accounts in domain
5. Set date range	Optional: set date range to capture full conversation history
6. Start export	Click Export → Google Vault queues the job (duration depends on data volume)
7. Download	When complete, download the export ZIP from Vault → transfer to migration machine

3.4 Vault Export Structure
Google Vault produces a ZIP archive structured per user. The migration tool reads this structure automatically — no manual mapping or users.json config required:

vault_export/
  ├── alice@company.com/
  │     └── gemini_conversations.json
  ├── bob@company.com/
  │     └── gemini_conversations.json
  ├── carol@company.com/
  │     └── gemini_conversations.json
  └── export_metadata.json

The tool reads export_metadata.json to discover all users, then processes each user's gemini_conversations.json in isolation. The user's email address is derived directly from the directory structure — no manual configuration needed.

3.5 Vault Export Contents
Data Type	Included in Vault Gemini Export
Gemini chat conversation history	Yes — all conversations within the specified date range
User prompts	Yes — full text of every prompt the user sent
Gemini responses	Yes — full text of every Gemini response
Conversation IDs	Yes — used to construct Gemini conversation URLs
Conversation timestamps	Yes — original dates preserved in export
Conversation titles	Yes — Gemini-generated titles for each conversation
AI-generated images (in-chat)	No — images are not stored or exportable from Vault
Rendered charts (in-chat)	No — visual renders are not stored; code that generated them is included if present in response text
Uploaded files	No — files uploaded to Gemini are processed temporarily and not retained

4. Deployment Architecture
4.1 End-to-End Admin Flow
Zero user involvement	Users are never contacted, notified, or asked to perform any action during this migration. The entire process runs between the IT admin, Google Vault, and Microsoft Graph API.

Step	Action
1. Vault export	Admin exports all users' Gemini history from Google Vault in one operation — downloads export ZIP
2. Configure tool	Admin sets API credentials in .env file — Microsoft Graph OAuth token and Azure OpenAI key
3. Dry run	Admin runs: migrate --vault ./vault_export --dry-run — reviews per-user summary and total cost estimate
4. Visual scan	Tool auto-runs visual asset scanner per user — admin reviews pre-migration report for flagged conversations
5. Run migration	Admin runs: migrate --vault ./vault_export — tool processes all users sequentially
6. Per-user processing	Tool auto-discovers each user from Vault structure → parses → replays prompts → creates Copilot Pages in that user's account → clears memory → next user
7. Report	Tool produces migration_report.json — admin reviews per-user page counts, errors, and flagged assets
8. Users log in	Each user opens Copilot and sees their migrated conversations under 'Recent pages' — no action needed from them

4.2 Per-User Data Isolation
Guarantee	Detail
User auto-discovery	Tool reads user list from Vault export_metadata.json — no manual config file needed
Export isolation	Each user's gemini_conversations.json is loaded, processed, and cleared from memory before the next user begins
Pages scoped to owner	Tool calls POST /users/{email}/drive/root/children — pages created in the specific user's account only
Admin visibility	Admin sees only the migration log: email, page count, status, duration, errors. Never sees page content
Default page privacy	All created pages are private to the user — standard M365 permissions apply
No user credentials	Tool uses admin's delegated Graph API token — no individual user passwords or tokens required
Memory clearing	Each user's conversation data, prompts, and responses are cleared from memory after their pages are created

4.3 CLI Interface
The migration tool is a single CLI command with the following flags:

migrate --vault <path>              Run full migration from Vault export directory
migrate --vault <path> --dry-run    Preview per-user summary and cost, no API calls
migrate --vault <path> --user <email>  Process one user only
migrate --vault <path> --resume     Skip users already in checkpoint file
migrate --vault <path> --parallel 3 Process up to 3 users concurrently
migrate --vault <path> --from-date 2024-01-01  Filter conversations by date

6. Functional Requirements
6.1 Module 1 — Vault Export Reader & User Discovery
ID	Requirement
FR-1.1	Accept a path to an unzipped Google Vault export directory as the sole input: --vault <path>
FR-1.2	Read export_metadata.json to auto-discover all users in the export — no manual user config required
FR-1.3	Validate that a gemini_conversations.json file exists for each discovered user — skip and log missing users
FR-1.4	Parse each user's JSON: extract conversation ID, title, timestamp, user prompts, and Gemini responses
FR-1.5	Construct Gemini conversation URL from conversation ID: https://gemini.google.com/app/{id}
FR-1.6	Support --from-date and --to-date flags to filter conversations by date range across all users
FR-1.7	Output auto-discovered user list with conversation counts before any processing begins

6.2 Module 2 — Visual Asset Scanner (per user)
ID	Requirement
FR-2.1	Scan each user's conversations for responses referencing images or visual content using heuristic keyword matching
FR-2.2	Scan for conversations containing chart/graph code (Python matplotlib, Chart.js, D3, SQL) in response text
FR-2.3	Generate a pre-migration visual asset report per user: flagged conversations with title, date, and Gemini URL
FR-2.4	Display warning: flagged conversations may contain images — these must be downloaded from Gemini before migration as they are not in the Vault export
FR-2.5	Migration proceeds automatically after scan — admin can review the report but no manual confirmation gate is required
FR-2.6	Pages for flagged conversations display a red warning banner with the Gemini conversation URL for post-migration image retrieval

6.3 Module 3 — Copilot Response Generator (per user)
ID	Requirement
FR-3.1	Primary: replay each user's prompts via Microsoft Graph API Copilot interactions endpoint
FR-3.2	Fallback: use Azure OpenAI API (GPT-4o) when Graph API is unavailable or admin approval is pending
FR-3.3	Send each prompt as a stateless standalone request — no conversation threading between prompts
FR-3.4	Flag follow-up prompts (e.g. 'make it shorter', 'continue') — include with a context prefix or skip based on --skip-followups flag
FR-3.5	Implement exponential backoff on rate limit (429) and server error (5xx) responses
FR-3.6	Write per-user checkpoint after each successful response — interrupted runs resume from last prompt
FR-3.7	Configurable concurrency per user (default: 3 parallel prompt requests, max: 10)
FR-3.8	Prompt and response data held in memory only — never written to any intermediate disk file

6.4 Module 4 — Copilot Pages Creator (per user) — Core Feature
ID	Requirement
FR-4.1	Authenticate using admin's delegated Graph API token, scoped to each user's account via /users/{email}/ path
FR-4.2	Create one Copilot Page per Gemini conversation: POST /users/{email}/drive/root/children
FR-4.3	Page title = original Gemini conversation title from Vault export
FR-4.4	Metadata header on each page: original conversation date, prompt count, direct Gemini conversation URL link
FR-4.5	Visual asset warning banner at top of page if conversation was flagged — includes Gemini URL for image download
FR-4.6	Per prompt structure: [Prompt block] → [Copilot response block] → [Gemini reference toggle] → [Notes block]
FR-4.7	Gemini reference toggle: expands inline to show archived Gemini response text from Vault export
FR-4.8	Gemini URL link inside toggle: 'Open original Gemini conversation' hyperlink to https://gemini.google.com/app/{id}
FR-4.9	Notes block: empty editable section per prompt — pre-labelled 'Add your notes here'
FR-4.10	Pages must appear in user's Copilot sidebar under 'Recent pages' within 60 seconds of creation
FR-4.11	After all pages for a user are created, clear that user's data from memory before processing the next user

6.5 Module 5 — Admin Migration Report
ID	Requirement
FR-5.1	Write migration_report.json on completion of all users
FR-5.2	Per-user row: email, conversations processed, pages created, prompts replayed, visual assets flagged, errors, duration
FR-5.3	Summary row: total users, total pages, total errors, total estimated API cost, total duration
FR-5.4	Error entries include: user email, conversation title, error message, and recommended action
FR-5.5	Report contains metadata only — zero conversation content, zero prompt text, zero response text
FR-5.6	Separate visual_assets_report.json listing per-user flagged conversations with Gemini URLs for admin reference

6.6 Module 6 — Google Drive → OneDrive Document Migration (per user)
ID	Requirement
FR-6.1	Use admin's Google service account with domain-wide delegation to access each user's Google Drive
FR-6.2	List Drive files for each user from the Gemini usage period via Google Drive API v3
FR-6.3	Download files in Office format: Docs → .docx, Sheets → .xlsx, Slides → .pptx
FR-6.4	Upload converted files to each user's OneDrive via Graph API: PUT /users/{email}/drive/root:/{path}:/content
FR-6.5	Flag Sheets files containing charts — note in report that charts survive .xlsx conversion
FR-6.6	Per-user document migration report: files moved, skipped, conversion errors, total size transferred

7. Copilot Page Structure
7.1 Page Layout
Block	Content
Page title	Original Gemini conversation title (from Vault export)
Metadata header	Original conversation date · Prompt count · 'Open in Gemini' hyperlink
Visual asset warning	Red banner — shown only if visual assets were flagged. Includes Gemini URL for manual image download
Prompt block (repeated)	Original user question — locked reference block, not editable
Copilot response block	Fresh Copilot response generated during migration — locked reference block
Gemini reference toggle	'View Gemini's original answer' — expands inline to show Vault-archived response
Gemini URL link	Inside toggle: 'Open original Gemini conversation' — links to https://gemini.google.com/app/{id}
Notes block	Empty editable section per prompt — user adds annotations post-migration
Ask Copilot bar	Persistent input at bottom of every page for follow-up questions

7.2 Gemini Reference Toggle Behaviour
Condition	Behaviour
Gemini URL recoverable	Toggle expands Vault-archived response text + shows hyperlink to open original Gemini conversation
No Gemini URL recoverable	Toggle expands response text + shows original date only — no hyperlink
Responses differ significantly	Orange 'Responses differ' badge shown inside the Gemini panel
Visual asset flagged	Red warning banner at top of page with Gemini URL for manual image download
Chart code in Vault response	Green 'Chart code — re-runnable' badge inside Gemini panel

8. Technical Architecture
8.1 System Flow
Stage	Detail

10. Security & Data Governance
Critical	User conversation data must never be written to any shared location, logged, or retained after the user's migration is complete. Each user's data exists in memory only during their processing window.

10.1 Data Handling Rules
Rule	Implementation
Vault export files	Stored only on admin's migration machine — never uploaded to any cloud service by the tool
Conversation data in processing	Held in memory only — never written to any intermediate disk file
API credentials	Stored in .env file only — never in code, CLI arguments, logs, or reports
Migration report content	Metadata only: counts, statuses, durations, error codes — zero conversation content
Checkpoint file content	Conversation IDs and completion booleans only — no prompt or response text
Admin access to pages	Delegated token used only to create pages — admin cannot read or modify page content after creation
Memory clearing	Each user's full dataset (prompts, responses, page content) cleared from memory after their pages are created
Vault export cleanup	Admin should delete the Vault export from the migration machine after successful migration and report sign-off

10.2 Required Permission Scopes
Permission	Type	Purpose
Files.ReadWrite.All	Delegated M365	Create Copilot Pages in each user's OneDrive
Sites.ReadWrite.All	Delegated M365	Write Loop/Pages content to user's M365 workspace
User.Read.All	Delegated M365	Resolve user email to M365 account for routing
Copilot.ReadWrite	Delegated M365	Replay prompts via M365 Copilot API (if available)
https://www.googleapis.com/auth/drive.readonly	Google service account	Read and export user Drive files for document migration

12. Known Limitations
Limitation	Mitigation
Vault not included in all Workspace plans	Verify Vault access before project starts — upgrade to Business Plus or Enterprise Standard if needed
Pages appear under 'Recent pages', not 'Recent chats'	Closest achievable experience — Microsoft provides no chat history write API
Pages show migration date, not original Gemini date	Original date preserved as metadata in the page header from Vault export
Copilot responses differ from Gemini originals	Gemini reference link lets users compare both responses — difference is expected and useful
AI-generated images not in Vault export	Visual asset scanner flags affected conversations with Gemini URLs — admin can retrieve post-migration
Gemini URL expires if user's Google account is deleted	Migrated pages retain the Gemini response text from Vault — URL is optional access path
Graph API admin approval takes 3–5 days	Azure OpenAI fallback available immediately with zero admin dependency
Vault retention policy may exclude old conversations	Check and extend retention window before export if historical data is needed
Follow-up prompts lack standalone context	--skip-followups flag excludes them; or include with auto-generated context prefix
